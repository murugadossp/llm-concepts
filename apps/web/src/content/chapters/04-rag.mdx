This chapter explores how language models leverage external memory and databases to answer questions about proprietary files, private codebases, or real-time news.

<Callout title="Learning goal">
  By the end of this module, you should be able to explain the mechanics of Retrieval-Augmented Generation (RAG), text chunking, vector indexing, hybrid keyword search, and the distinction between short-term and long-term agent memory.
</Callout>

## Core Roadmap

To build a retrieval-augmented system, we break the problem into four areas:

1. **Knowledge Cutoffs & Prompts**: Why static pre-trained weights need external facts at runtime.
2. **Chunking & Vector DBs**: How long books are split into snippets and indexed in databases.
3. **Hybrid Search & Re-ranking**: Combining classic word matches with vector angles, and sorting results.
4. **Agent Memory Types**: How agents use local context, databases, and user summaries to behave consistently.

<RememberCard>
  - Training weights = static, baked-in memory.
  - RAG = runtime lookup, dynamic memory.
  - Vector DBs = semantic filing cabinets indexable by Cosine Similarity.
</RememberCard>
