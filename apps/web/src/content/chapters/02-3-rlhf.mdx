Even after SFT, a model can still produce harmful, toxic, or factually incorrect statements. To prevent this, models pass through **alignment** using human preference feedback.

## Quick Summary

- RLHF guides models toward responses that humans prefer.
- DPO bypasses training a separate reward model by mathematically optimizing directly on preferences.
- Constitutional AI uses AI feedback (RLAIF) to align models against a set of written principles.

---

## The Traditional RLHF Pipeline

Reinforcement Learning from Human Feedback (<abbr title="Reinforcement Learning from Human Feedback">RLHF</abbr>) is a three-stage loop:

```
[Prompt] ──> Generate Pair (A, B) ──> Human Ranks (A > B) ──> Train Reward Model ──> Update Policy (PPO)
```

1. **Preference Data Collection**: The model generates two responses (A and B) for a given prompt. Human labelers review them and mark which one is better (e.g. A is more polite, accurate, or safe than B).
2. **Train a Reward Model**: We train a separate neural network called the **Reward Model**. This model's sole job is to read a prompt and a response, and output a single score representing how much a human would like that response.
3. **Optimize the Policy (PPO)**: We use an algorithm called **Proximal Policy Optimization (<abbr title="Proximal Policy Optimization">PPO</abbr>)** to nudge the base model's weights. If the model outputs tokens that get high scores from the Reward Model, those weights are strengthened. If it outputs tokens that get low scores, those weights are suppressed.

### The Reward Hacking & KL-Divergence Barrier
During reinforcement learning, models are prone to **reward hacking**—discovering bizarre gibberish strings that trigger high scores from the reward model without actually being helpful. 
To prevent this, we subtract a **Kullback-Leibler (KL) divergence penalty** from the reward function:

$$\text{Reward}_{\text{total}} = R(x, y) - \beta D_{\text{KL}}(P_{\theta}(y|x) \parallel P_{\text{ref}}(y|x))$$

Where:
* $R(x, y)$ is the raw reward model score.
* $P_{\theta}(y|x)$ is the active model being updated.
* $P_{\text{ref}}(y|x)$ is the original frozen SFT model.
* $\beta$ is a scaling hyperparameter (typically 0.1). If the model's token distribution drifts too far from the safe SFT base, the KL divergence penalty increases, keeping the model stable.

## PPO vs. DPO: Bypassing the Reward Model

PPO is notoriously complex to implement. It requires hosting four separate models in GPU memory simultaneously: the active policy, the frozen reference, the reward model, and the value critic model.

![PPO vs DPO Training Loops](/images/ch2-training/02-3-ppo-vs-dpo-loops.jpg)

In 2023, researchers introduced **Direct Preference Optimization (<abbr title="Direct Preference Optimization">DPO</abbr>)**:
* DPO mathematically demonstrates that the active language model can act as its own reward model.
* Instead of training a separate reward model and using reinforcement learning, DPO updates the model weights directly using a simple binary cross-entropy classification loss on the preference pairs (chosen vs. rejected).
* This makes alignment faster, cheaper, and far more stable.

## Constitutional AI: RLAIF

As models grow, collecting millions of human annotations becomes a massive bottleneck. Anthropic introduced **Constitutional AI (or Reinforcement Learning from AI Feedback - RLAIF)**:
* **The Constitution**: Developers write a list of principles (e.g., "Do not support illegal acts", "Be supportive but direct").
* **Self-Critique**: The model generates an answer, reads the constitution, critiques its own draft, and writes an updated version.
* **AI Preferences**: Another model acts as the judge, ranking response pairs based on the constitution. This preference data is then used to train the final model.

<ELI5Card title="An editor with a code of conduct">
  Instead of asking a crowd of people to vote on every single sentence a writer writes, the publisher gives the writer a written handbook (the Constitution). The writer checks their own work against the handbook before publishing.
</ELI5Card>

  - Constitutional AI replaces human annotators with structured rules and AI feedback.
