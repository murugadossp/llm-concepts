Attention is the idea that turns a row of token vectors into context-aware representations.

## Quick Summary

- Attention lets tokens look at other tokens in the same context.
- Query, key, and value are the classic ingredients.
- Attention blends information; it does not simply pick one perfect source.

## The Transformer: A Repeating Prediction Machine

Most modern LLMs use a transformer architecture. At a high level, a transformer repeats the same kind of block many times:

1. Turn token IDs into embeddings.
2. Let tokens exchange information with attention.
3. Pass the result through a multilayer perceptron, a learned network inside each block.
4. Repeat for many layers.
5. Produce logits, which are scores for possible next tokens.
6. Sample or choose the next token.

<ELI5Card title="A stack of reading passes">
  The first pass notices nearby words. Later passes combine bigger ideas. By the end, the model has a context-aware representation it can use to guess the next token.
</ELI5Card>

## Everyone Asks Everyone

Self-attention allows tokens in a sequence to dynamically exchange information. Instead of just reading words from left to right, every token inspects all other tokens in the context window and calculates a relevance weight.

In plain English, self-attention asks: *"For this token, which other tokens in the prompt should influence what I mean right now?"*

To calculate this, the transformer converts each token's embedding into three separate vectors:
1. **Query (Q)**: What the token is looking for (an inquiry).
2. **Key (K)**: What a token offers to help matches (an index label).
3. **Value (V)**: The actual content or meaning that is passed along if there is a match.

<Callout variant="tip" title="Analogy: The Library Card Catalog">
  Imagine you are in a library looking for books about **solar energy**.
  * Your search query is **"solar energy"** (the **Query**).
  * You walk to the card catalog and check the index labels on the drawers. One label says **"green power"** (the **Key**).
  * Since your Query matches that Key, you pull out the cards inside the drawer.
  * The actual books on the shelves that you take home to read are the **Value**.
  
  In an LLM, the model multiplies the Query vector of one word with the Key vectors of all other words. The higher the score, the more of the Value vector it copies over to update its own meaning!
</Callout>

## Multi-Head Attention: Many Perspectives

A model doesn't just run this search once per layer. It runs it multiple times in parallel, which is called **Multi-Head Attention**.
For example, a model might have 12 or 32 attention "heads." Each head projects the tokens into a different subspace, allowing it to focus on a different type of relationship:
* **Head 1 (Grammar)**: Focuses on matching verbs with their nouns (e.g. connecting "sat" with "cat").
* **Head 2 (Pronouns)**: Focuses on resolving pronouns (e.g. connecting "it" with "mat" or "cat").
* **Head 3 (Sentiment/Tone)**: Focuses on adjectives and tone modifiers.

By combining the outputs of all these heads, the model builds a rich, multi-layered understanding of the sentence simultaneously.

<ELI5Card title="A team of expert readers">
  Instead of one person reading a book, imagine a team of 12 specialists. One person highlights verbs, another looks for subjects, and a third tracks pronouns. At the end of the page, they merge their notes to get a complete, multi-layered summary.
</ELI5Card>

## A Tiny Example

In the sentence "The cat sat on the mat because it was tired," the token `it` may need information from `cat` to make sense. Attention gives the model a way to make that connection, even when the related words are not next to each other.

Real attention patterns are not always this clean. A token can attend to punctuation, grammar cues, repeated phrases, or earlier instructions. The key idea is that meaning is updated by context.

<DeepDive title="Softmax behind attention">
  In scaled dot-product attention, the model compares a query vector with key vectors using dot products. The scores are scaled, passed through softmax, and used as weights over value vectors.

  Softmax turns raw scores into a distribution. The highest-scoring tokens get more weight, but lower-scoring tokens can still contribute.
</DeepDive>

<DeepDive title="Why positional encoding exists">
  Attention by itself does not know word order. Without position information, the model would struggle to distinguish "dog bites man" from "man bites dog."

  Rotary Position Embedding is one common technique. It rotates parts of the query and key vectors based on position, which helps the model compare relative positions during attention.
</DeepDive>
