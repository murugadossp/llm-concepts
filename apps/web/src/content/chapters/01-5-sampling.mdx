The model does not write a whole answer in one motion. It predicts one token, then another, then another.

## TL;DR

- Logits are raw scores for possible next tokens.
- Temperature changes how sharp or adventurous the choice can be.
- Top-p and top-k limit the candidate set before the next token is chosen.

## From Scores To Text

Once the model has scores for possible next tokens, decoding rules decide what comes out.

Temperature changes how sharp or flat the probability distribution is. Low temperature makes the model favor high-probability tokens. Higher temperature gives lower-probability tokens more room.

Top-p sampling, also called nucleus sampling, keeps the smallest set of likely tokens whose probabilities add up to a chosen threshold. Top-k sampling keeps only the k most likely tokens.

## Why The Same Prompt Can Differ

If a model has several plausible next tokens, sampling can choose different paths. After one different token, the next probability menu changes too. That is why small randomness early in an answer can lead to a noticeably different final response.

Stable tasks usually use lower randomness. Brainstorming and creative tasks can benefit from more room to explore.

## From Logits to Probabilities: The Softmax Math

Before an LLM picks the next token, the final layer of the network outputs a list of raw, unnormalized scores called **logits** (usually ranging from negative to positive infinity). 

To convert these raw numbers into a tidy list of percentages that sum to 1.0 (100%), the model uses the **Softmax** function. Mathematically, the probability of selecting token $i$ is calculated as:

\[P(x_i) = \frac\{e^\{z_i / T\}\}\{\sum_j e^\{z_j / T\}\}\]

Where:
* $z_i$ is the raw logit score for token $i$.
* $T$ is the **Temperature** scaling factor.
* $e$ is Euler's constant ($\approx 2.718$).

### How Temperature $T$ Controls the Math:
* **Low Temperature ($T \to 0$)**: The divisor $T$ is very small, which pushes the exponential term $e^\{z_i/T\}$ of the highest-scoring token toward infinity. The highest-scoring token takes up virtually 100% of the probability, making generation completely predictable (greedy decoding).
* **High Temperature ($T \ge 1.0$)**: The divisor $T$ is large, which shrinks the differences between the exponents. The probability curve flattens out, giving lower-scoring words a much higher relative chance of being rolled.

## Developer Cheat Sheet: Sampling Configurations

When writing API requests to models like Gemini, GPT, or Claude, you can adjust these sampling values. Here is a practical cheat sheet:

| Task Type | Temperature | Top-P | Top-K | Best Used For |
| :--- | :--- | :--- | :--- | :--- |
| **Strict / Deterministic** | `0.0` | `1.0` | `1` | Code generation, JSON schema validation, math, factual lookup. |
| **Standard Q&A / Chat** | `0.7` | `0.9` | `40` | Customer support, email drafting, summarizing. |
| **Creative / Brainstorming** | `1.2` | `0.95` | `80` | Story writing, naming products, advertising taglines. |
