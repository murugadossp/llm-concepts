This module slows the foundation down. Instead of rushing through the entire transformer stack, each lesson teaches one idea, gives it a visual model, then explains where the metaphor stops working.

<Callout title="Learning goal">
  By the end of this module, you should be able to explain how text becomes tokens, how tokens become vectors, how attention mixes context, why context windows matter, and how sampling changes model output.
</Callout>

## Start Here

0. **What Is an LLM, Really?** Understand next-token prediction over a trained model.
1. **What Is a Token?** Learn the pieces an LLM can actually read.
2. **Embeddings** See how token IDs become coordinates in meaning-space.
3. **Attention** Understand how tokens decide which other tokens matter.
4. **Context Windows** Learn what fits on the model's working page.
5. **Sampling** Explore why the same prompt can produce different answers.
6. **Putting It Together** Trace the full transformer pipeline cycle.

<RememberCard>
  - Token = chunk of text the model sees.
  - Embedding = learned vector for a token.
  - Attention = context-aware information mixing.
  - Context window = the model's working page.
  - Sampling = the rules for choosing the next token.
</RememberCard>

## Why This Module Matters

Agents, retrieval systems, tool calls, memory, evaluations, and safety all depend on these basics. If tokens feel fuzzy, token budgets feel arbitrary. If embeddings feel magical, retrieval feels magical. If attention feels vague, long-context behavior feels unpredictable.

The point is not to memorize architecture diagrams. The point is to build a working mental model you can use when a real LLM app behaves strangely.
