Now that you have learned about tokens, embeddings, attention, context, and sampling, it is time to connect all these pieces in a single lifecycle.

## Quick Summary

- Text generation is a closed-loop system: Output token is fed back as input.
- Tokenizers convert text; embeddings map position; attention mixes meaning; sampling rolls the dice.
- Every single output token requires running the entire network once.

## The Lifecycle of a Token

Every time a language model generates a single word, it executes a clean, structured cycle:

1. **Input Stage**: The model looks at the active text (system prompt + history + user query).
2. **Tokenizer Stage**: Raw characters are chopped into token IDs.
3. **Embedding Stage**: The IDs are looked up as coordinate vectors in meaning-space.
4. **Attention Stage**: Tokens inspect and mix information with one another, making the representations context-aware.
5. **Logits Stage**: The final layers score all potential words in the model's vocabulary.
6. **Sampling Stage**: Creative dials filter candidates and roll the dice to select the next token.

This new token is immediately appended to the input context, and the whole cycle starts over.

## Remember

<RememberCard>
  - **Tokens** are the vocabulary chunks the model works with.
  - **Embeddings** represent tokens geometrically to encode baseline meaning.
  - **Attention** shares information between tokens to build context-aware vectors.
  - **Context Windows** set the size limit of the model's active working memory.
  - **Sampling** dials control randomness and choice selection.
  - **The Loop** feeds selected tokens back to predict subsequent words.
</RememberCard>
