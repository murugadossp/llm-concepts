This chapter details the mechanics of **Function Calling**, which bridges raw text prediction and active code execution.

<Callout title="Learning goal">
  By the end of this module, you should be able to:
  * **Design schemas**: Declare parameters, types, and descriptions in JSON schema format.
  * **Trace the loop**: Coordinate model function-calls with backend API executors.
  * **Execute parallel calls**: Fan out and resolve multiple queries concurrently.
  * **Scale catalogs**: Deploy semantic routing to handle hundreds of tools without context bloat.
</Callout>

## Core Roadmap

To build system integrations with function calling, we divide the topic into three lessons:

1. **Schemas & Descriptions**: Designing parameter constraints and convincing descriptions.
2. **The Execution Loop**: Tracing how code executes and fanning out parallel calls.
3. **Routing & Scaling**: Scaling tool selection and managing crowded catalogs.

<RememberCard>
  - A tool is a function definition declared to the LLM.
  - Models never call APIs directly; they emit a structured request to your server.
  - Good tool descriptions act as documentation the model reads.
</RememberCard>
