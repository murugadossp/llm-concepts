Chapter 09 · Advanced coordination

When agents build shared understanding.

The earlier chapters showed how agents route work. This chapter shows what happens when agents collaborate more deeply: they build shared state, reason over each other's findings, adapt roles, and generate a custom workflow for each complex request.

Collaborative stateDynamic rolesAdaptive planning5 quiz questions
1 Learning objectives

Advanced coordination is more than routing.

Basic coordination asks, "Who should act next?" Advanced coordination asks, "What do we understand together, who should lead now, and how should the workflow change?"

1 Shared understanding

Explain how agents use shared state to build a common picture instead of isolated updates.

2 Dynamic roles

Apply role adaptation when a primary agent should become a specialist, lead, or coordinator.

3 Adaptive systems

Design workflows that improve coordination by recording what worked and what caused friction.

2 Collaborative research process

A useful collaboration has a shape.

Imagine a customer asks: "Should we refund this enterprise account, replace the device, or offer service credit?" Multiple agents need to research, contribute, challenge, and synthesize.

Teaching point

The shared context is the anchor. Without it, agents may produce individually reasonable answers that do not combine into one reliable system answer.

3 Collaborative state building

Each agent adds a layer to the shared case file.

Collaborative state is not a pile of random notes. It is an evolving structure where every agent contributes a specific kind of evidence.

Order agentfacts
Writes: purchase date, delivery status, warranty, prior contacts.
Quality agentevidence
Writes: defect likelihood, photo confidence, replacement risk.
Policy agentconstraints
Writes: refund eligibility, exception rules, approval threshold.
Retention agentrelationship
Writes: customer value, churn risk, goodwill recommendation.
Risk

Shared state helps agents align, but it can also become too large. Keep state fields typed, named, and owned. "Everyone writes everything" is how collaboration turns into confusion.

4 Dynamic role adaptation

Roles are defaults, not cages.

In an advanced system, an agent can shift behavior when the situation changes. The response agent might become an escalation coordinator. The quality agent might become a specialist lead if defect evidence dominates the case.

Role rule

Role adaptation should be visible in state. Record why the role changed, who now owns the next decision, and when control returns.

5 Adaptive workflow generation

The graph can choose the right team for the request.

Adaptive planning does not mean improvising without structure. It means the graph assesses the request, identifies needed expertise, then selects a bounded workflow.

Simple request

Assess complexitylow
Needed expertiseorder + policy
Workfloworder → policy → response

Complex request

Assess complexityhigh
Needed expertisequality + retention + approval
Workflowquality → retention → coordinator → response
def choose_workflow(state: RefundState) -> list[str]:
    agents = ["order_agent", "policy_agent"]
    if state["defect_likelihood"] > 0.7:
        agents.append("quality_agent")
    if state["churn_risk"] > 0.6:
        agents.append("retention_agent")
    if state["approval_required"]:
        agents.append("human_review")
    agents.append("response_agent")
    return agents
