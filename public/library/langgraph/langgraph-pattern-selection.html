Chapter 10 · Summary

Choose the pattern by the coordination need.

Coordinator centralizes communication, specialist teams group agents behind interfaces, and event-driven collaboration lets agents respond as new facts appear. Real systems often combine all three.

Decision guideTrade-offs6 quiz questions
1 Selection rule

Start with the question your workflow is asking.

1 Is there one clear sequence?

Use a coordinator. It understands the overall workflow and manages dependencies.

2 Are capabilities naturally grouped?

Use specialist teams. Each team coordinates internally and exposes a clean interface.

3 Should agents wake up only when something changes?

Use event-driven collaboration. Agents subscribe to events that matter to them.

1B Complexity check

Watch for the N-squared communication problem.

A fully connected agent network looks flexible, but it becomes hard to reason about quickly. If every agent can hand work to every other agent, possible handoffs grow as n * (n - 1).

3 agents

6 possible directed handoffs. Still understandable.

6 agents

30 possible directed handoffs. Testing gets noisy.

10 agents

90 possible directed handoffs. You need structure.

Selection hint

If communication links are exploding, add a coordinator, group agents into teams, or switch to event subscriptions so agents only react to what matters.

2B Production readiness

Before calling it enterprise-ready, check the boring parts.

The pattern gets you the shape. Production needs guardrails around that shape so teams can operate, debug, and improve the system.

1 State schema

Define the fields every agent can read and write. Validate updates so one bad node cannot corrupt the case file.

2 Agent boundaries

Give each agent a clear job, input contract, output contract, and stop condition.

3 Persistence

Checkpoint important runs so failures, reviews, and retries can resume from known state.

4 Monitoring and tracing

Record which node ran, what it changed, why it routed, and which fallback path was used.

5 Error handling

Add retries, fallbacks, circuit breakers, and graceful degradation for APIs and tools.

6 Human review

Pause for approval on high-value refunds, legal risk, customer escalation, or irreversible actions.

Testing hint

Use mocked model outputs, edge-case states, and canary releases before sending real customer cases through the graph.

3 Related patterns

Other patterns you will see in real systems.

These are useful, but they are better taught after the main three are clear.

Router

A lightweight classifier chooses the next agent or path.

Evaluator or critic

One agent produces; another reviews quality, policy, or completeness.

Planner-executor

A planner breaks the task into steps; executors perform them.

Map-reduce

Many agents process chunks in parallel; one reducer combines results.

Human-in-the-loop

The graph pauses for human approval before sensitive actions.

Hierarchical

A top-level coordinator manages sub-coordinators and their teams.
